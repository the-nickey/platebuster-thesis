# platebuster — open-source web service for license plate blurring

> 🇬🇧 English version. 🇷🇺 [Russian (primary)](README.md).

Comparative study of license plate detection approaches (classical CV vs neural networks across architecture families — CNN, attention-centric, transformer-based) and an open-source web service that hides plates on photos with proper perspective handling via 4-corner homography.

**License:** [AGPL-3.0](LICENSE) — required by Ultralytics YOLO. See [`THIRD_PARTY_NOTICES.md`](THIRD_PARTY_NOTICES.md).

## What's inside

- Six models compared in one UI: classical baseline (contours + morphology), ResNet18-regression, YOLO11n-detect + ResNet18-keypoint, YOLO11n-pose, YOLO12-pose, RF-DETR Nano/Small/Medium.
- 4-corner homography — plates are masked with proper perspective, not as flat rectangles.
- Two modes: blur or logo overlay.
- Two builds: full local one (6 models, comparison mode) and a slim one for Streamlit Community Cloud (single model, up to 10 photos at a time).

## Stack

- **Models:** YOLO11/YOLO26-pose (Ultralytics, AGPL-3.0), RF-DETR (Apache-2.0), ResNet18-regression baseline, classical baseline.
- **Training:** PyTorch 2.4+ with MPS (Apple Silicon) and CUDA in cloud.
- **Annotation:** CVAT (see [`infra/cvat/`](infra/cvat/)) + lightweight annotators in [`scripts/annotation/`](scripts/annotation/).
- **Web:** Streamlit.

## Quick start — local app

```bash
# from repo root
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# download weights into weights/ — see weights/README.md

# full build (6 models)
streamlit run streamlit_app/app.py --server.port 8600
```

On macOS you can also double-click [`platebuster-full.command`](platebuster-full.command) (full) or [`platebuster-cloud.command`](platebuster-cloud.command) (slim) instead of running the command.

## Reproducing the experiment from scratch

```bash
pip install -r requirements-dev.txt

# 1. Datasets — see data/README.md
python scripts/download_ccpd.py
python scripts/build_openalpr_dataset.py
python scripts/download_roboflow.py        # needs ROBOFLOW_API_KEY in .env
python scripts/build_unified_dataset.py

# 2. Training
python scripts/training/train_yolo_pose.py
python scripts/training/train_yolo_detect.py
python scripts/training/train_rfdetr.py
python scripts/training/train_keypoint_head.py

# 3. Evaluation
python scripts/training/eval_classical.py
python scripts/training/eval_two_stage_pipeline.py
python scripts/benchmark_latency.py
python scripts/collect_results.py
```

## What's NOT in the repo and why

- **Datasets** (`data/`) — tens of gigabytes of public data with various licenses. Downloaded by scripts, not redistributed.
- **Weights** (`weights/`, `*.pt`, `*.pth`) — published separately via Releases / Hugging Face Hub. See [`weights/README.md`](weights/README.md).
- **CVAT sources** — that's a separate open-source project; here you'll only find instructions, see [`infra/cvat/README.md`](infra/cvat/README.md).
- **Training artefacts** (`runs/`, outputs of `qc_*.py` etc.) — regenerated by scripts.
